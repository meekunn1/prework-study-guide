HTML

  • The head element contains information about the webpage.
  • Meta data is child element inside the head element (parent element)
  • Metadata is data is used by browser to serve our webpage to users.
  • The body element represents the visible content shown to the user.

CSS

  • How to link .CSS to .html: In "head", after the final "meta" and before "title" element, add: link rel="name1" href="name2" (all inside < >).
    • the "link" element creates the link
    • "name1" for "rel" attribute specifies the relationship between the current document and linked document resource.
    • "name2" for "href" attribute specifies location of the external resource. Location is derived from the location of the .html location.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Class attributes: in CSS, indicated by "." followed by name for class selector.

Git

  • git status: checks what branch we are currently on
  • git pull origin main = get latest version of the project on your local computer
  • git work flow for working locally.
    1. Prepare remote work: get to repo on Github. press "code" button and copy SSH
    2. go to Gitbash. navigate to desired folder. enter: git clone "paste SSH here" = download repo content in current directory
    3. Start work flow: git checkout -b "enter new branch name" = create and change to working branch
      • checkout = move the working branch to a new branch
      • -b = create a new branch
    4. git status = to double-check you are working in the new branch and not in the main branch
    5. Get coding: work and make changes
    6. Complete the work flow: git add -A = Commit all files in the working branch.
      • add = add modifications in the current working branch to the staging area.
      • -A = indicates that we want to add all changes.
    7. git commit -m "insert message here" = associate message to commit.
    8. "git pull origin main" to check for message: Already up to date.
    9. git push origin "insert feature name" = upload changes to github
    10. go on github and get you your working branch under repo
    11. create pull request
